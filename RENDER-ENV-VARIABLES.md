## üìã How to Add Environment Variables in Render

1. Go to your service in Render dashboard
2. Click **"Environment"** tab on the left
3. Click **"Add Environment Variable"**
4. Copy-paste from sections below
5. Click **"Save Changes"**

## üöÄ Phase 1: Landing Page (Minimum Required)

**Add these NOW to get your landing page working:**

```

Value: https://creai.onrender.com
```

```

Value: production
```

**That's it!** Your landing page will work with just these two.

## üóÑÔ∏è Database (Auto-Generated by Render)

**You don't need to add this manually!**

When you create a PostgreSQL database in Render and link it to your web service, Render automatically adds:

```
Key: DATABASE_URL
Value: postgresql://user:password@hostname:5432/database
```

**Important:** Use the **Internal Database URL** for better performance and security.

## üîí Security Best Practices

### ‚úÖ DO:
- Use **different** API keys for development vs production
- Rotate secrets every 90 days
- Use **test mode** keys until you're ready to go live
- Keep secrets in password manager (1Password, Bitwarden)
- Revoke unused API keys

### ‚ùå DON'T:
- Commit secrets to Git (ever!)
- Share secrets in Slack/Email
- Use production keys in development
- Leave old API keys active
- Copy secrets to multiple places

---

## üö® If You Leaked a Secret

**Act immediately:**

1. **Revoke the compromised key** (in the service's dashboard)
2. **Generate a new key**
3. **Update Render environment variable**
4. **Trigger a redeploy** (to use new key)
5. **Monitor for unauthorized usage**

**Services to check:**
- GitHub: Search your repos for accidentally committed secrets
- Git history: Check if secret was ever committed
- Logs: Review application logs

## üí° Pro Tips

### Organize Variables

Use prefixes to group related variables:

```
# Database
DATABASE_URL=...
DATABASE_POOL_SIZE=10

# Stripe
STRIPE_SECRET_KEY=...
STRIPE_PUBLISHABLE_KEY=...
STRIPE_WEBHOOK_SECRET=...

# Email
SENDGRID_API_KEY=...
SENDGRID_FROM_EMAIL=...
```

### Use Render's Built-in Features

Render can auto-generate certain values:

```yaml
envVars:
  - key: NEXTAUTH_SECRET
    generateValue: true  # Render creates random value
```

### Environment-Specific URLs

```
# Development
NEXT_PUBLIC_APP_URL=http://localhost:3000

# Staging
NEXT_PUBLIC_APP_URL=https://creai-staging.onrender.com

# Production
NEXT_PUBLIC_APP_URL=https://www.creai.dev
```

## üÜò Troubleshooting

### "Environment variable not found"

1. Check spelling (case-sensitive!)
2. Make sure you clicked "Save Changes"
3. Wait for redeploy to complete (~2-3 min)
4. Check in "Environment" tab that variable exists

### "Invalid API key"

1. Verify key is correct (no extra spaces)
2. Check you're using correct environment (test vs production)
3. Verify key hasn't been revoked
4. Regenerate key if needed

### "Build fails after adding variables"

1. Check for syntax errors in environment variable values
2. Make sure required variables are all present
3. Check build logs for specific error
4. Remove recently added variables to isolate issue
